# Summary Outputs

This directory contains paper-ready tables generated by `summarize_outputs.py`.

## Files Generated

### Main Tables

1. **`table_main_regression.csv`** - Main coefficient table (wide format with separate Coef/SE columns)
   - Best for: Excel, Google Sheets, manual editing
   - Format: Each model has two columns: `ModelName_Coef` and `ModelName_SE`
   - Significance stars: *** p<0.001, ** p<0.01, * p<0.05

2. **`table_main_regression_combined.csv`** - Main coefficient table (combined format)
   - Best for: Quick viewing, LaTeX conversion
   - Format: `coefficient*** (SE)` in single column per model
   - Example: `212.85*** (31.00)`

3. **`table_main_regression_long.csv`** - Long format table
   - Best for: Data analysis, merging with other datasets
   - Format: One row per variable-model combination

### Robustness Tables

4. **`supp_robustness_summary.csv`** - Robustness summary across all specifications
   - Columns:
     - `Model`: Model name and type
     - `Flood_Sign`: Sign of flood coefficient (+/-)
     - `Flood_Sig`: Significance stars for flood coefficient
     - `Waste_Sign`: Sign of waste coefficient (+/-)
     - `Waste_Sig`: Significance stars for waste coefficient
     - `Pop_Sign`: Sign of population coefficient (+/-)
     - `Pop_Sig`: Significance stars for population coefficient
     - `Adj_R2`: Adjusted R² (or pseudo R² for SLM)
     - `Moran_I`: Moran's I statistic
     - `Moran_P`: Moran's I p-value

### Metadata

5. **`outputs_manifest.md`** - Complete manifest of all outputs
   - Documents which files correspond to which tables/figures
   - Useful for responding to reviewers
   - Lists all source files used

## Usage

Run the summary script after running all model scripts:

```bash
python summarize_outputs.py
```

The script will:
1. Load all coefficient files from `Output/flood/`, `Output/wasteflood/`, and `Output/wastefloodpop/`
2. Create merged coefficient tables
3. Generate robustness summary tables
4. Create outputs manifest

## Notes

- The script automatically handles different file naming conventions (e.g., `flood_only_*` vs regular names, FILE_ID suffixes)
- Missing values are left blank in tables
- All significance tests use p<0.05 threshold
- For flood-only models, Adj R² may need manual verification from OLS summary files

## Customization

To customize the output:
- Edit `summarize_outputs.py` to change significance thresholds
- Modify `format_coefficient()` function to change coefficient formatting
- Adjust `create_robustness_summary()` to add/remove summary columns
